Descrição:
Este simples comando visa facilitar a construção de um item por processo de crafting.
Ele pode ser chamado por um menu,  uma function ou um trigger de item.

Acima está listado todos as características possíveis a serem transmitidas para o item
que está sendo criado pelo processo. Porém não é obrigatório o uso de TODOS.
Os obrigatórios são:

obj=<src.findid.i_faz_item.uid>
obj.cleartags
obj.tag.delay
obj.tag.skill
obj.tag.dif
obj.tag.res1
obj.tag.nam1
obj.tag.amt1
obj.tag.item
fazitem

Flags especiais:

WMAKEFUNR - Will Make Furniture.
Após a construção do item, quando ele for dclicado um menu aparecerá perguntando se o 
jogador que invocou o dclick quer ser o dono do item ou não.
Qualquer item pode virar movel deste jeito. Tanto uma cama, um armário ou uma espada.

WPN - Weapon
PARA '"TODAS"' AS ARMAS E ARMADURAS E SOMENTE ELAS.
Quando o item é construido, o valor de seus HITPOINTS atuais e máximos são calculados atravez
da skill ARMSLORE do fabricante e guardados nas variáveis MORE1L e MORE1H do item
fazendo a arma/armadura 100% compatível com o sistema de combate do shard.
Negligenciar este dado fará com que a arma/armadura fique com vida infinita.

DONO - Ser dono do item como movel
(não aconselhavel na maioria dos casos. Use WMAKEFURN).
Assim que i tem for construido ele será adicionado ao sistema de moveis e o fabricante será automaticamente o dono.

Funcionalidades diversas:

obj.tag.neartype é um atributo muito interesante e inovador, pois sem o uso de TARGET ou qualquer outro
trigger testa se o jogador está a uma distância máxima de 2 tiles de qualquer item que tenha este TYPE.
Note que por ser um novo comando HardCodded (IsNearType <type> <dist>) este atributo só funcionará com
um TYPE hardcoded do Sphere e não funcionará para nenhum [TYPEDEF] criado via script.

obj.tag.name agora aceita comando %s para o fato do amount do item ser maior que 1. Criando então
o efeito de 'a flecha' ou '35 fechas' sem outra instervenção de script.

Os personagens não ficam mais freeze enquanto trabalham. Porem eles ganham um envento e_fazitem.
Enquanto eles estão com este evento, se andarem ou invocarem um @dclick em qualquer objeto resulta
em uma falha automática, anulando também o script de fazer mais de um item por vez (explicado adiante).

Skill e testes:
Quando você seleciona um valor para o atributo obj.tag.skill, você obriga o sistema a testar aquela skill contra a
dificuldade obj.tag.dif quando o item termina de ser feito.
O teste de sucesso não é mais baseado em funções RANDOM mas sim pela função SKILLCHECK <skill> <valor>
hardcoded do novo Sphere. Isso faz com que não seja impossível um usuário com skill baixa consiga um sucesso
num teste alto, mas deixa muito improvável. O teste de sucesso está muito parecido com o antigo padrão do UO.
Quando você for acertar o valor de obj.tag.diff, tenha em mente que este formato de skill nunca foi utilizado antes.
Não usa-se 100.0, nem 1000 mas simplesmente 100.
Ou seja, se você quiser que um teste tenha como alvo 23.8 use simplesmente 24.
O sistema é progressivo logarímo, fazendo com que quanto maior a atual skill dele, mais demorado será subi-la.
Mas ela sempre VAI subir. Não existe um meio de uma skill não subir por vários erros sucessivos em umt este RANDOM
pois o sistema não é baseado em funções RANDOM.

Fazendo mais de um item por vez:
O usuário pode repetir o precesso automaticamente até dez vezes.
Após tentar fazer uma unidade do item desejado, ele pode utilizar o comando ".fazitem <num>" para repetir o ultimo processo
<num> vezes. Este comando .fazitem é [PLEVEL 1].
